- Claude Code に日本語で指示する際のコツ
- 日本語出力を引き出すプロンプトの書き方
- 日本語環境での注意点とよくある問題
- CLAUDE.md に日本語設定を書く方法

## Claude Code の日本語対応について

Claude Code は日本語を含む多言語に対応しています。指示を日本語で入力すれば、
基本的に日本語で回答してくれます。ただし、**いくつかの注意点**を押さえておくと、
より精度の高い出力が得られます。

実際に日本語で使い続けてわかったことを共有します。

## 日本語で指示するときのコツ

### 1. 具体的・明示的に書く

英語と同様、曖昧な指示では期待通りの出力が得られないことがあります。

```
# 曖昧な指示（避ける）
「このコードを直してください」

# 具体的な指示（推奨）
「src/api/user.ts の getUserById 関数で、IDが存在しない場合に null を返すのではなく
 UserNotFoundError をスローするよう修正してください」
```

### 2. 出力言語を明示する

コードのコメントやドキュメントを日本語にしたい場合は明示します：

```
「以下の関数に JSDoc コメントを追加してください。コメントは日本語で書いてください」
```

### 3. 文末に「〜してください」形式を使う

命令形より依頼形の方が自然で、Claude Code も安定して処理します：

```
# 命令形（動くが若干不自然）
「ファイルを読む。バグを探す。修正する。」

# 依頼形（推奨）
「ファイルを読んで、バグを探し、修正してください」
```

## CLAUDE.md で日本語設定を固定する

プロジェクトの CLAUDE.md に言語設定を書いておくと、毎回指示しなくて済みます：

```markdown
## 言語設定

- コードのコメントは日本語で書くこと
- エラーメッセージ・ログは英語のまま維持すること
- ドキュメント・README は日本語で記述すること
- Git コミットメッセージは英語で書くこと（例: feat: add user authentication）
```

これを設定しておくと、指示ごとに「日本語で」と書かなくて済みます。

## よくある日本語の問題と対処法

### 問題1: コメントが英語で出力される

Claudeはデフォルトで英語コメントを好む傾向があります。

**対処法**: CLAUDE.md または都度指示で言語を指定する。

```
「追加するコメントは日本語で書いてください」
```

### 問題2: 文字化けが発生する

Windows 環境で日本語が文字化けする場合があります。

**対処法**: ターミナルのエンコーディングを UTF-8 に設定する。

```bash
# Windows PowerShell
chcp 65001

# または Windows Terminal を使用（デフォルトで UTF-8）
```

### 問題3: 長文の日本語指示が途中で切れる

非常に長い日本語の指示はトークンを消費します。

**対処法**: 指示を分割するか、箇条書きで簡潔にまとめる。

```
# 長文（避ける）
「このプロジェクトは〇〇という目的で作られており、△△という技術を使っていて、
 ××というコーディング規約があって...（長文続く）...修正してください」

# 箇条書き（推奨）
「以下の条件で src/components/Button.tsx を修正してください：
- disabled 状態のスタイルを追加
- aria-disabled 属性を設定
- TypeScript の型を Button | 'submit' | 'reset' に変更」
```

## 日本語プロンプトのテンプレート

よく使う指示のテンプレートをまとめておきます：

```bash
# バグ修正
claude "[ファイルパス] を読んで、[具体的なバグの症状] を修正してください。修正箇所にコメント（日本語）を追加してください"

# 機能追加
claude "[ファイルパス] に [機能の説明] を追加してください。既存のコードスタイルに合わせてください"

# コードレビュー
claude "[ファイルパス] をレビューして、改善点があれば日本語で説明してください。修正まで行わず、提案のみにしてください"

# ドキュメント生成
claude "[ファイルパス] の全関数に JSDoc コメントを追加してください。説明は日本語で書いてください"
```

## まとめ

Claude Code を日本語で使うポイントは以下の3つです：

1. **CLAUDE.md に言語設定を書く**（毎回指示不要になる）
2. **具体的・明示的な指示を心がける**（曖昧な指示は精度が下がる）
3. **箇条書きで構造化する**（長文は分割する）

英語と比べると多少の精度差はありますが、日常的な開発タスクであれば十分な品質で動作します。
